## 3. AI 모델 평가 [상세]

### (V) 완료 — 260504

| 항목 | 내용 |
|------|------|
| 학습 데이터 | Nikto 기반 실제 공격 로그 450건, 3개 피처 |
| 하이퍼파라미터 | contamination=0.05 (IQR 분석 기준), n_estimators=100 |
| 평가 결과 | benchmark.py로 데이터 품질 진단, 이상 탐지 비율 55.6% |
| Weighted FedAvg | 3노드 가중치 집계 (IDC 60% / 지사 24.9% / 소규모 15.1%) 시뮬레이션 성공 |
| sklearn 1.8.0 패치 | `_decision_path_lengths` Private 속성 → `getattr` 재구성 로직 해결 |
| Privacy-Preserving | 원본 로그 미전송, Tree 구조/Offset 파라미터만 전송 |

**기술 선택 근거 (면접 답변)**
- IF 선택: Autoencoder(리소스 과다) / One-Class SVM(대규모 느림) 대비, 비지도 + 실시간 스코어링 최적
- contamination=0.05: 비운영 환경 공격 밀도 기반, 운영 시 0.01~0.05 하향 예정
- n_estimators=100: 50개 대비 안정적, 200개 이상 대비 추론 속도 우세

**잔여 부족한 점**
- Sentinel 값(국가코드 99 등)이 Recall 지표 왜곡 → Data Cleaning 파이프라인 미구축
- Shannon Entropy Feature 미도입 → 피처 3→4개 확장 필요
- 모델 한계 미기술 → "Low-and-Slow 공격에 취약" 명시 필요

**잔여 개선 액션**
1. `calculate_entropy()` 추가 (`train_model.py`, `analyze_anomalies.py`)
2. Sentinel 값 필터링 전처리 단계 추가
3. README 모델 한계 섹션 추가
