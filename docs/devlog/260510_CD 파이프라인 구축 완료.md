## 3. 트러블슈팅

### TS-1. tfplan 파일 Job 간 전달 실패
**증상**: `tfplan: no such file or directory`  
**원인**: `terraform plan -out=tfplan` 결과 파일을 별도 step에서 참조 시 경로 문제  
**해결**: `-out=tfplan` 제거, plan/apply 동일 Job 내 연속 실행 + `apply -auto-approve`

### TS-2. DynamoDB state lock 권한 없음
**증상**: `AccessDeniedException: dynamodb:PutItem/GetItem`  
**원인**: `github_actions_minimal_policy`에 DynamoDB 권한 누락  
**해결**: `dynamodb:GetItem`, `PutItem`, `DeleteItem` + S3 tfstate 버킷 권한 추가

### TS-3. IAM 정책이 Role에 미연결
**증상**: 권한 추가 후에도 동일 에러 반복  
**원인**: `github_actions_minimal_policy`에 `aws_iam_role_policy_attachment` 리소스 자체가 없었음  
**해결**: `github_actions_attach` attachment 리소스 추가 후 로컬 `terraform apply` → push

### TS-4. 전체 인프라 권한 부족
**증상**: IAM, API Gateway, ACM, Route53, Budgets, S3 등 다수 서비스 403  
**원인**: minimal policy가 EC2/VPC 권한만 포함. Terraform plan은 모든 리소스 Read 필요  
**해결**: `AdministratorAccess` managed policy 추가 (OIDC trust policy가 레포 단위로 제한)

### TS-5. 하드코딩 로컬 경로
**증상**: `mkdir /home/march: permission denied`  
**원인**: `archive_file` data source의 `source_dir`, `output_path`가 `/home/march/...` 절대경로  
**해결**: `${path.root}/../lambda/...` 상대경로로 교체 (lambda.tf, edge_security.tf, traffic_generator.tf)

### TS-6. Lambda 함수명 불일치
**증상**: `ResourceNotFoundException: Function not found: devsecops-security-analyzer`  
**원인**: workflow에서 임의로 작성한 함수명과 실제 AWS 배포 함수명 상이  
**해결**: Terraform state 실제 ID 기준으로 수정

| workflow (수정 전) | AWS 실제 함수명 |
|---|---|
| `devsecops-security-analyzer` | `SecurityAnalyzer` |
| `devsecops-security-preventer` | `SecurityPreventer` |
| `devsecops-normal-traffic-generator` | `NormalTrafficGenerator` |
