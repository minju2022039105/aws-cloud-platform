  [수정 필요] — 진행했으나 오류 또는 방향 불일치

  ① GitHub Actions 워크플로우 SSH 배포
  동적 IP 화이트리스팅 + 멱등성 처리 코드까지 작성했으나 Exit    
  Code 254/128로 실제 동작 미검증 상태. 개선안 5번 "CI/CD 보안   
  게이트" 관점에서 배포 파이프라인 자체가 불안정한 채로 남아있음.

  ② variables.tf 하드코딩 제거 미완성
  account_id, my_ip만 분리됐고 KMS ARN, WAF IPSet ARN, S3 버킷명,
   EC2 인스턴스 타입 등 다수가 모듈 코드에 하드코딩된 채로 잔존. 
  개선안의 "환경별 분리 가능한 구조" 요구사항 절반 미달.

  ③ EC2 인스턴스 태그 미검증
  IAM 정책에 Project: devsecops-platform 태그 Condition을        
  걸었으나 EC2 인스턴스에 실제로 해당 태그가 붙어있는지 미확인.  
  태그 없으면 Terminate/Stop/Start 권한이 사실상 작동 안 함 —    
  보안 설계가 공허해지는 상황.

  [개선 가능] — 구현됐으나 고도화 여지

  ① OIDC Condition StringLike → StringEquals
  현재 레포의 모든 브랜치/PR에서 Role Assume 가능. 배포는        
  main에서만 하므로:
  # 현재
  StringLike = { "...sub" =
  "repo:minju2022039105/aws-devsecops-platform:*" }
  # 고도화
  StringEquals = { "...sub" = "repo:minju2022039105/aws-devsecops
  -platform:ref:refs/heads/main" }

  ② Terraform provider 버전 미고정
  provider.tf에 >= 1.0만 있음. 개선안의 "버전 관리" 항목. ~> 6.0 
  형태로 고정하면 예기치 않은 breaking change 방지.

  ③ Permission Boundary 미설정
  SCS 관점에서 IAM Role에 Permission Boundary를 추가하면 정책이  
  아무리 넓어도 최대 권한 상한선이 강제됨. 지금 minimal policy   
  자체가 좁긴 하지만, 추후 정책이 느슨해질 리스크까지 봉쇄하는   
  방어적 설계.
