## 5. 트러블슈팅

### TS-1. WAF가 HTTP API(v2)와 연결 불가

**증상**: `aws_wafv2_web_acl_association` apply 시 `WAFInvalidParameterException` — ARN 형식 오류
```
arn:aws:apigateway:us-east-1::/apis/auzes14y53/stages/$default
```
**원인**: AWS WAF Regional은 REST API(v1) 스테이지만 지원. HTTP API(v2)의 `/apis/` ARN 형식은 허용하지 않음
**해결**: `aws_apigatewayv2_*` → `aws_api_gateway_*` (REST API v1)로 전환

---

### TS-2. REST API Stage CloudWatch 로깅 권한 오류

**증상**: `BadRequestException: CloudWatch Logs role ARN must be set in account settings`
**원인**: REST API Stage 로깅은 계정 레벨 IAM 역할이 사전 설정되어 있어야 함
**해결**: `apigateway.tf`에 추가
```hcl
resource "aws_iam_role" "apigw_cloudwatch" { ... }
resource "aws_api_gateway_account" "main" {
  cloudwatch_role_arn = aws_iam_role.apigw_cloudwatch.arn
}
```
`aws_api_gateway_stage`에 `depends_on = [aws_api_gateway_account.main]` 추가

---

### TS-3. 커스텀 도메인 AlreadyExists 충돌

**증상**: `BadRequestException: The domain name you provided already exists`
**원인**: 이전 apply 실패 과정에서 REST API v1 도메인(`aws_api_gateway_domain_name`)이 AWS에 생성됐지만 Terraform state에 저장되지 않음 → state에 없으니 재생성 시도 → 충돌

**확인**:
```bash
# v2 도메인: NotFound (이미 삭제됨)
aws apigatewayv2 get-domain-name --domain-name minju-devsec.store

# v1 도메인 고아 리소스 존재 확인
aws apigateway get-domain-name --domain-name minju-devsec.store --region us-east-1
```

**해결 시퀀스**:
```bash
# 1. v1 도메인이 AWS에 존재하면 state로 import
terraform import aws_api_gateway_domain_name.main minju-devsec.store

# 2. 불필요한 yourdomain.com 호스팅 영역 삭제 (사이드이펙트 없음)
aws route53 delete-hosted-zone --id <zone-id>

# 3. Route53 구 A 레코드 import (삭제 후 재생성하면 state 불일치 발생)
terraform import 'aws_route53_record.api' 'Z04690263FQ7USBL2HWDX_minju-devsec.store_A'

# 4. apply
terraform apply
```

**교훈**: apply 실패 후 AWS에 생긴 고아 리소스는 `terraform import`로 state에 편입시킨 뒤 apply 재시도. 수동 삭제 후 재생성은 Route53 레코드 충돌 등 2차 에러 유발 가능.
