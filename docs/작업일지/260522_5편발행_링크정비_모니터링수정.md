## 1. monitor.py 피처명 수정

재학습된 모델(`isolation_forest_model.pkl`)이 `path_entropy` + `args_entropy` 피처를 사용하는데 `monitor.py`는 `uri_entropy`를 생성하고 있어 `scaler.transform()` 단계에서 `ValueError` 발생.

### 수정 내용

- `FEATURES` 리스트: `uri_entropy` → `path_entropy`, `args_entropy`
- 데이터 로드 시 피처 생성: `df["uri_entropy"]` → `df["path_entropy"]`, `df["args_entropy"]`
- 시나리오 모드별 피처 주입: `PREDICT`, `PREMITIGATE`, `ATTACK_ATTEMPT` 모드 전부 수정

## 2. KMS 키 정책 invalid principal 수정

### 문제

```
MalformedPolicyDocumentException: Policy contains a statement with one or more invalid principals.
with module.security.aws_kms_key.waf_s3_key, on modules/waf/main.tf line 73
```

CI/CD GitHub Actions에서 Terraform apply 중 KMS 키 정책 업데이트 실패.

### 원인

`Enable Admin Privilege` Statement에 존재하지 않는 IAM 유저 ARN이 포함되어 있었음.

```hcl
# 수정 전
Principal = {
  AWS = [
    "arn:aws:iam::${var.account_id}:root",
    "arn:aws:iam::${var.account_id}:user/system/devsecops-admin-user"
  ]
}
```

KMS 키 정책은 S3 버킷 정책과 달리 **존재하지 않는 principal을 검증**함.

### 수정

```hcl
# 수정 후
Principal = {
  AWS = "arn:aws:iam::${var.account_id}:root"
}
```

## 3. AWS 액세스 키 재발급 및 자격증명 업데이트

- AWS 콘솔에서 액세스 키 재발급
- 로컬 `~/.aws/credentials` → `aws configure`로 업데이트
- GitHub Actions는 `AWS_ROLE_ARN` 기반 OIDC 방식 → 액세스 키 불필요, CI/CD 영향 없음
- GitHub Secret `WAF_IPSET_ARN` 현재 실제 ARN으로 업데이트

## 4. Athena DB 불일치 수정

### 문제

`aiops_results` 테이블이 `monitoring_db`에 있었고 Lambda/Grafana는 `default.aiops_results`를 참조 중 → No data.

### 해결

`default` DB에 `aiops_results` 테이블 생성 + `MSCK REPAIR TABLE` 실행.

현재 Athena 테이블 정리:

| 테이블 | DB | 용도 |
|---|---|---|
| `aiops_results` | default | SOAR Lambda + Grafana AI 대시보드 |
| `model_metrics` | default | Grafana 모델 성능 KPI |
| `waf_logs_minju_project` | default | 미사용 (삭제 가능) |
| `waf_access_logs` | monitoring_db | Grafana WAF 대시보드 |
