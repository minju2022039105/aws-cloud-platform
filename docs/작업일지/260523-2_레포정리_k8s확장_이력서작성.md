## 2. kubernetes-extension/ 디렉토리 생성

기존 서버리스 포트폴리오 메시지 유지하면서 Kubernetes 역량 보완용 미니 프로젝트 디렉토리 구성.

```
kubernetes-extension/
├── ai-inference-server/
│   ├── app/
│   │   ├── main.py        # FastAPI /health, /predict 엔드포인트
│   │   ├── model_loader.py
│   │   └── schemas.py
│   ├── Dockerfile         # 빌드 컨텍스트 레포 루트 기준
│   └── requirements.txt
├── k8s/
│   ├── deployment.yaml    # liveness/readiness probe 포함
│   ├── service.yaml
│   ├── configmap.yaml
│   └── hpa.yaml           # CPU 70%, 1~5 replicas
├── helm/ai-inference/
│   ├── Chart.yaml
│   ├── values.yaml
│   └── templates/         # deployment, service, hpa, _helpers
└── README.md
```

**Dockerfile 모델 포함 방식**
- 빌드 컨텍스트를 레포 루트로 설정 (`docker build -f ... .`)
- `ai/models/*.pkl` → 컨테이너 내부 `/models/`로 복사
- `.dockerignore` 추가: `.venv/`, `infra/`, `docs/` 등 제외, `ai/models/*.pkl` 명시적 허용
